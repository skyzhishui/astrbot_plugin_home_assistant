# Home Assistant 智能家居控制插件

> 一款用于 AstrBot 的 Home Assistant 智能家居控制插件，支持灯光和开关设备的控制。

## ✨ 功能特性

- 💡 **灯光控制**：开关灯、调节亮度
- 🔌 **开关控制**：控制插座、电器等开关设备
- 📋 **设备查询**：查看所有智能设备状态
- 🤖 **LLM 工具**：支持 AI 主动控制智能家居
- ⚙️ **灵活配置**：可独立开关各项功能

## 📦 安装

将插件文件夹放入 AstrBot 的插件目录：

```
/AstrBot/data/plugins/astrbot_plugin_home_assistant/
```

然后重启 AstrBot 或在 WebUI 中加载插件。

## ⚙️ 配置项说明

在 AstrBot WebUI 的插件配置页面可修改以下选项：

### 基础配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `ha_url` | `http://127.0.0.1:8123` | Home Assistant Web 访问地址 |
| `ha_token` | *(预设值)* | 长期访问令牌，用于 API 认证 |
| `timeout` | `10` | API 请求超时时间（秒） |

### 功能开关

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `enable_light_commands` | `true` | 启用灯光控制命令 |
| `enable_switch_commands` | `true` | 启用开关控制命令 |
| `enable_llm_tools` | `true` | 启用 LLM 工具，允许 AI 主动控制设备 |

---

## 📝 配置项行为描述

### `ha_url` - Home Assistant 地址

**作用**：指定 Home Assistant 服务的 Web 访问地址，插件通过此地址与 HA 通信。

**行为**：
- 插件初始化时会使用此地址连接 Home Assistant
- 支持 IP 地址或域名，需包含端口号（如非 80/443）
- 如果地址不可达，所有设备操作都会失败

**示例**：
- `http://192.168.1.100:8123`
- `https://homeassistant.local:8123`

---

### `ha_token` - 长期访问令牌

**作用**：用于身份验证，让插件有权限操作 Home Assistant 中的设备。

**行为**：
- 所有 API 请求都会携带此令牌
- 令牌无效时，所有操作都会失败

**获取方式**：
1. 登录 Home Assistant Web 界面
2. 点击左下角用户头像 → 安全
3. 滚动到底部「长期访问令牌」
4. 点击「创建令牌」并复制

---

### `timeout` - 请求超时

**作用**：控制与 Home Assistant API 通信的最大等待时间。

**行为**：
- 默认 10 秒，如果 HA 响应较慢可增大此值
- 超时后操作会失败并返回错误

---

### `enable_light_commands` - 灯光命令开关

**作用**：控制是否启用灯光相关的用户命令。

**开启时**：
- `/灯光列表` - 查看所有灯光设备
- `/开灯 <名称>` - 打开灯光
- `/关灯 <名称>` - 关闭灯光
- `/调光 <名称> <亮度>` - 调节亮度

**关闭时**：
- 以上命令仍可触发，但会返回「💡 灯光控制命令已禁用」
- LLM 工具不受影响（由 `enable_llm_tools` 单独控制）

---

### `enable_switch_commands` - 开关命令开关

**作用**：控制是否启用开关相关的用户命令。

**开启时**：
- `/开关列表` - 查看所有开关设备
- `/打开 <名称>` - 打开开关
- `/关闭 <名称>` - 关闭开关

**关闭时**：
- 以上命令仍可触发，但会返回「🔌 开关控制命令已禁用」
- LLM 工具不受影响（由 `enable_llm_tools` 单独控制）

---

### `enable_llm_tools` - LLM 工具开关

**作用**：控制是否允许 AI（大语言模型）主动调用智能家居控制工具。

**开启时**：AI 可以通过自然语言指令控制设备
```
用户: 帮我把客厅灯打开
AI: 调用 homeassistant_turn_on_light → "已成功打开 客厅灯"

用户: 家里有哪些灯开着？
AI: 调用 homeassistant_get_lights → 返回灯光状态列表

用户: 把书房插座关掉
AI: 调用 homeassistant_turn_off_switch → "已成功关闭 书房插座"
```

**关闭时**：AI 无法主动控制设备
```
用户: 帮我把客厅灯打开
AI: 请使用命令 /开灯 客厅 来控制灯光
```

**注意**：此开关只影响 AI 工具调用，不影响用户命令（命令由 `enable_light_commands` 和 `enable_switch_commands` 控制）。

### 灯光相关

| 工具名 | 功能 | 触发场景示例 |

## 🔧 支持的设备类型

| 类型 | 实体前缀 | 说明 |
|------|----------|------|
| 灯光 | `light.*` | 所有灯光设备 |
| 开关 | `switch.*` | 普通开关、智能插座等 |
| 虚拟开关 | `input_boolean.*` | Home Assistant 虚拟开关 |

## 🐛 故障排除

### 命令返回「已禁用」
- 检查对应的命令开关是否开启（`enable_light_commands` 或 `enable_switch_commands`）

### 设备列表为空
1. 确认 Home Assistant 地址和令牌正确
2. 确认网络连接正常
3. 确认 Home Assistant 中有对应类型的设备

### LLM 工具不生效
1. 确认 `enable_llm_tools` 已开启
2. 确认设备名称与 Home Assistant 中的名称一致
